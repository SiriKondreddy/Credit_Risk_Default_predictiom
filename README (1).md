# Credit Risk Default Prediction

Machine learning case study that predicts the probability of a loan applicant defaulting, using the [Credit Risk Dataset](https://www.kaggle.com/datasets/laotse/credit-risk-dataset). Built as part of the **Innovexa Catalyst** case study.

> **Target variable:** `loan_status` (`1` = default, `0` = non-default)

## Overview

The notebook walks through a full credit-risk modeling pipeline:

- **EDA** — structure checks, missing values, duplicates, implausible values (e.g. age outliers), univariate/bivariate/multivariate analysis, and target imbalance (~22% default vs. ~78% non-default).
- **Feature engineering** — affordability and leverage ratios, behavioral/stability ratios, ordinal encoding of loan grade, a prior-default flag, income brackets, and a high-risk interaction feature.
- **Modeling** — baselines (Logistic Regression, Decision Tree) followed by Random Forest, XGBoost, LightGBM, and CatBoost, with class imbalance handled via **SMOTE** and **class weighting**, and hyperparameter tuning via **Optuna**.
- **Evaluation** — AUC-ROC, F1, Precision, Recall, PR-AUC, and the KS statistic, plus ROC/PR curves, a confusion matrix, 5-fold stratified cross-validation, and stability checks across loan-grade and home-ownership segments.
- **Interpretation** — global and local explanations with **SHAP**, a decile score-band analysis to validate rank-ordering of risk, and a reusable scoring function for new applicants.

## Final Model

An **Optuna-tuned LightGBM** model is selected as the final model based on AUC-ROC, F1, PR-AUC, KS statistic, and stability across customer segments. The trained model and its expected feature columns are persisted with `joblib` for reuse.

## Dataset

The notebook expects a `credit_risk_dataset.csv` file in the working directory with (at minimum) the following columns:

| Column | Description |
|---|---|
| `person_age` | Applicant age |
| `person_income` | Annual income |
| `person_home_ownership` | RENT / MORTGAGE / OWN / OTHER |
| `person_emp_length` | Employment length (years) |
| `loan_intent` | EDUCATION / MEDICAL / VENTURE / PERSONAL / DEBTCONSOLIDATION / HOMEIMPROVEMENT |
| `loan_grade` | Credit grade, A (best) to G (worst) |
| `loan_amnt` | Loan amount |
| `loan_int_rate` | Loan interest rate |
| `loan_percent_income` | Loan amount as a percentage of income |
| `cb_person_default_on_file` | Y/N — prior default on file |
| `cb_person_cred_hist_length` | Length of credit history (years) |
| `loan_status` | Target — 1 = default, 0 = non-default |

Download it from [Kaggle](https://www.kaggle.com/datasets/laotse/credit-risk-dataset) and place it in the project root before running the notebook.

## Scoring a New Applicant

The notebook includes a `score_applicant()` function for scoring a single new loan applicant against the trained model:

```python
new_customer = {
    'person_age': 28,
    'person_income': 55000,
    'person_home_ownership': 'RENT',
    'person_emp_length': 4.0,
    'loan_intent': 'EDUCATION',
    'loan_grade': 'B',
    'loan_amnt': 10000,
    'loan_int_rate': 11.5,
    'loan_percent_income': 0.18,
    'cb_person_default_on_file': 'N',
    'cb_person_cred_hist_length': 5,
}

score_applicant(new_customer)
```

This returns the predicted default probability, predicted class, a risk tier (D1–D10 decile bands), and the top SHAP-based drivers behind the prediction.

## Evaluation Metrics

Given the class imbalance, model selection prioritizes:

- **AUC-ROC** — overall discriminative power
- **PR-AUC** — performance on the minority (default) class
- **F1-Score / Precision / Recall** — classification quality at the chosen threshold
- **KS Statistic** — separation between default and non-default score distributions

Accuracy is intentionally de-emphasized as a primary metric, since it can be misleading on imbalanced data.
